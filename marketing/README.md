# Marketing Assets

Home for Chrome Web Store screenshots, promo tiles, launch graphics, and any other marketing materials for this extension.

## Recommended workflow

Use the Claude Code skill `/app-store-screenshots` to scaffold a Next.js + html-to-image project right here in this directory. Every asset becomes version-controlled JSX you can iterate on.

### First-time setup

From this directory, run in Claude Code:

```
/app-store-screenshots
```

Describe your extension when prompted. The skill generates React components that render at target dimensions and export to PNG.

### What to generate

| Asset | Dimensions | CWS-required? | Ship-by-default? |
|---|---|---|---|
| Store screenshot (hero / thumbnail) | 1280×800 | Yes | Yes |
| Additional store screenshots (up to 4 more) | 1280×800 | No | Yes (target 5 total) |
| Small promo tile | 440×280 | No | **Yes** |
| Marquee promo tile | 1400×560 | No | **Yes** |
| Social launch graphics (Twitter card / OG image) | 1200×628 or 1200×630 | No | Optional |

First screenshot = the thumbnail in search results. Make it the most compelling one.

**Why ship the promo tiles even though CWS doesn't require them.** Common wrong takeaway: *"marquee is only shown if Google features me, so I'll skip it."* This is backwards. Google's Featured-badge automation checks for a **complete listing with promotional images** as one of its criteria (along with MV3 manifest, user count, privacy-policy checkbox — see `docs/03-cws-best-practices.md` → Featured badge). No promo tile = automatically disqualified from Featured. Featured status drives almost every high-discoverability placement in the store. Each tile takes under a minute to generate from the same skill that made your screenshots. **Always ship both.** The only reason to skip is if you've decided not to market the extension.

### Directory layout (once scaffolded)

```
marketing/
├── README.md              (this file)
├── package.json           (generated by skill)
├── app/                   (Next.js app router)
│   └── screenshots/
│       ├── hero/          (one route per screenshot variant)
│       ├── feature-1/
│       └── ...
├── components/            (shared layout, browser chrome mockup, annotations)
├── assets/                (logos, app screenshots, source PNGs)
└── output/                (generated PNGs — gitignored)
```

### Fallback

If you don't use Claude Code, take screenshots manually at 1280×800 using CleanShot or your OS screenshotter, and save them to `.output/screenshots/`.

---

## Not part of the extension build

Everything in `marketing/` is **separate from the extension itself**. It has its own `package.json`, its own dependencies, and is never bundled into `.output/chrome-mv3/`. Think of it as a sibling tool that happens to live in the same repo.

Generated files (`node_modules/`, `.next/`, `output/`) are already gitignored.
