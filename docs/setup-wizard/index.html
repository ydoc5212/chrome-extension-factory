Chrome Web Store API — Setup

Step 1 of 5

Publishing to the Chrome Web Store via the API needs 4 credentials and a Google Cloud project. The factory script scripts/bootstrap-gcp.ts automates steps 1–2 (and attempts step 3). Step 4 — creating an OAuth Desktop client — has no public Google API, so you'll click through the console. This page walks you through it with exact buttons and direct links.

Configure OAuth consent screen

partially automated

The script tried to create an IAP OAuth brand for you. For personal Google accounts (non-Workspace), Google requires you to configure this in the console yourself. For Workspace accounts, it's probably already done — skip to step 4.

Show me the clicks
  1. Open the consent screen (link below).
  2. Click Get started.
  3. Enter any app name (e.g. CWS Upload) and your email as support contact.
  4. Audience: pick External if you're on a personal account, Internal if Workspace.
  5. Leave scopes empty — the refresh-token tool declares its own.
  6. Save. You don't need to "publish" the app; test mode is fine.

4

Create an OAuth 2.0 Client ID

manual — Google provides no API

This is the step that can't be automated. You'll click through the console and copy two values back to the terminal. Takes about 60 seconds.

  1. Open the Credentials page for your project (button below).
  2. Click + Create Credentials at the top, then OAuth client ID.
  3. Application type: Desktop app. Name it Chrome Webstore Upload.
  4. Click Create. A modal shows your Client ID and Client secret.
  5. Copy both, then paste them into the terminal when prompted.
Open Credentials page →

Why can't this be automated? The IAP OAuth Admin API was deprecated January 2025 and shut down March 2026. No public replacement exists for creating general-purpose Desktop OAuth clients. See docs/07-google-cloud-setup.md for the full story.

5

Generate a refresh token

automated

The skill runs npx chrome-webstore-upload-keys for you. It prompts for the Client ID + Secret from step 4, opens a browser for your consent, and prints a refresh token. The token is saved to .secrets.local.json automatically.
